# Матрица дизайн-инструментов Yalken v1

Статус: рекомендательный рабочий гайд

Проверено: 2026-07-20

Область: исследование, проектирование, реализация и шлифовка интерфейса Yalken

Эта матрица не является новым продуктовым каноном и не разрешает добавлять
сетевые зависимости в Yalken. Инструменты работают снаружи приложения: в
Codex, Figma, браузере или локальном CLI. Любое изменение продукта по-прежнему
подчиняется `CANON.md` и `docs/YALKEN_DESIGN_OS_CHANGE_GUIDE_V2_2.md`.

Полный исследовательский резерв, включая инструменты, которые сейчас не
устанавливаются, хранится в `YALKEN_DESIGN_TOOL_RESERVE_V1.md`. Резерв не
расширяет активный контур и используется только по зафиксированным триггерам.

## 0. Быстрый вход для нового агента

Эта матрица включается только для актуальной задачи интерфейса. Она не является
фоновым чек-листом и не требует запускать инструменты просто потому, что они
установлены.

| Проверка | Решение агента |
|---|---|
| Задача меняет продуктовый экран, сценарий, состояния, типографику, палитру, токены, отступы или визуальный эталон | поставить `DESIGN_TOOL_ROUTER: APPLIED`, назвать визуальный вопрос и продолжить по матрице |
| Задача относится к backend, CLI, infra, данным, тестам или обычной документации и не меняет интерфейс | поставить `DESIGN_TOOL_ROUTER: NOT_APPLICABLE` и не вызывать дизайн-инструменты |
| Правка механически возвращает интерфейс к уже утверждённому эталону | исследование не запускать; использовать только необходимую визуальную проверку результата |
| Неясно, меняется ли дизайн-контракт | свериться с каноном и задать владельцу один уточняющий вопрос до запуска инструментов |

Правило минимальности: один сформулированный вопрос — один основной инструмент —
один проверяемый результат. `ACTIVE` означает доступность, а не обязательность.
Резерв не открывается, пока текущая матрица закрывает задачу.

## 1. Статусы

| Статус | Значение |
|---|---|
| `ACTIVE` | Подключено и может использоваться в обычной задаче |
| `GATED` | Подключено, но вызывается только при выполнении условия |
| `COLD` | Настроено, но отключено до конкретной задачи |
| `RESTART_PENDING` | Подключено во внешней конфигурации, текущую сессию нужно перезапустить |
| `AUTH_PENDING` | Официальное подключение выбрано, но требует авторизации владельца |
| `SESSION_PENDING` | Плагин установлен, но локальный мост поднимается из открытой сессии приложения |
| `WEB` | Бесплатный браузерный справочник или ручной инструмент |
| `LATER` | Нужен на зрелом этапе, сейчас не устанавливается в проект |
| `REFERENCE_ONLY` | Изучаем паттерны, не копируем систему и не ставим зависимость |
| `EXCLUDED` | Осознанно не входит в текущий рабочий пул |

## 2. Матрица инструментов

| Инструмент | Статус | Цена | Задача | Когда вызывать |
|---|---|---:|---|---|
| Lazyweb | `ACTIVE` | бесплатно в текущем подключении | реальные продуктовые экраны и исследование на основе проверяемых примеров | перед созданием или заметным изменением продуктового экрана |
| A1 Gallery | `ACTIVE` | бесплатно в текущем подключении | визуальные направления, сайты и отдельные секции | когда нужны композиция, типографический характер или визуальный язык |
| Figma MCP | `ACTIVE` | текущий бесплатный доступ | чтение макетов, компоненты, переменные и перенос решений | когда есть Figma-файл или нужно проверить дизайн до кода |
| Penpot MCP | `SESSION_PENDING` | бесплатно | программная отрисовка и редактирование макетов | после открытия Penpot, запуска плагина и локального моста |
| Browser | `ACTIVE` | бесплатно | открыть первоисточник, проверить интерфейс и локальную сборку | для проверки фактов, поведения и визуального результата |
| Context7 | `ACTIVE` | бесплатно | актуальная документация библиотек | перед использованием API существующей зависимости |
| UI Craft, Audit, Heuristic, Finalize | `ACTIVE` | бесплатно | качество UI, доступность, эвристика и предрелизная проверка | на реализации, после реализации и перед закрытием контура |
| Leonardo MCP | `ACTIVE` | бесплатно, Apache 2.0 | контрастные темы, WCAG, APCA как дополнительный сигнал, OKLCH и палитры | при любом изменении палитры, семантических цветов или состояний |
| Project Wallace | `ACTIVE` | бесплатно, MIT | измеримые расхождения CSS: цвета, размеры, специфичность и дубли | после крупной CSS-правки и перед финальной шлифовкой |
| Asimonim MCP | `GATED` | бесплатно, GPLv3 | поиск, валидация и конвертация DTCG-токенов | только после появления одного утвержденного DTCG-источника |
| Component Gallery | `WEB` | бесплатно | сравнение одного компонента между зрелыми системами | перед новой таблицей, селектом, диалогом, навигацией или пустым состоянием |
| Open UI | `WEB` | бесплатно | семантика и ожидаемое поведение сложных веб-контролов | перед combobox, select, popover, dialog и похожими контролами |
| Harmonizer | `WEB` | бесплатно | построение согласованной OKLCH-палитры | при поиске характера новой палитры |
| Polychrom | `WEB` | бесплатно | ручная проверка APCA и OKLCH в Figma | при проверке текстовых пар и состояний в макете |
| 21st Reference | `COLD` | бесплатный доступ уже настроен | справочный код и варианты компонентов | только когда визуальный контракт уже выбран |
| Paper | `COLD` | бесплатно в доступной версии | быстрые визуальные эскизы | для отдельного исследовательского контура |
| OpenPencil | `RESTART_PENDING` | бесплатно | программная работа с `.fig`, анализ, lint и экспорт | после перезапуска Codex; когда нужен локальный редактируемый макет |
| SiteOne Crawler | `GATED` | бесплатно | карта разрешённого сайта, Markdown, снимки и технический аудит | когда нужен воспроизводимый слепок конкретного сайта |
| Impeccable detector | `GATED` | бесплатно | дополнительный детерминированный поиск UI-антипаттернов | как второй сигнал после рендера; результаты проверять вручную |
| MCP Inspector | `GATED` | бесплатно | изолированная проверка протокола и инструментов MCP | перед регистрацией нового стороннего MCP |
| Open Design | `EXCLUDED` | бесплатно | отдельная HTML/design-as-code лаборатория | не использовать: релиз `0.15.1` не прошёл строгую локальную проверку подписи |
| Playwright Reference | `COLD` | бесплатно | браузерная автоматизация и проверка размеров окна | для веб-поверхности или отдельного браузерного стенда |
| Argos Hobby | `LATER` | 5 000 снимков в месяц бесплатно | облачная визуальная регрессия и проверка | после появления стабильного набора экранов и детерминированных снимков |
| Blueprint, Cloudscape, PatternFly, Ant Design Pro | `REFERENCE_ONLY` | бесплатно как справочник | desktop-паттерны, плотность, состояния и клавиатурный UX | перед сложной desktop-поверхностью; код и стили не переносить |
| Northbase, Ripplix, Design Spells | `REFERENCE_ONLY` | есть бесплатный просмотр | состояния, микрокопирайтинг и микровзаимодействия | когда базовый сценарий уже понятен и нужна точечная шлифовка |
| Glyphs MCP, Fontra, Fontspector, Diffenator3 | `EXCLUDED` | смешанная | самостоятельное редактирование и сборка шрифтов | не использовать, пока Yalken работает с готовыми гарнитурами |

## 3. Рабочий маршрут

### Этап 0. Границы задачи

1. Прочитать канон и актуальный гайд Design OS.
2. Выбрать одну группу изменения и один визуальный вопрос.
3. Зафиксировать, что является истиной: утвержденный скрин, Figma или решение владельца.

### Этап 1. Исследование

1. Начать с Lazyweb для реальных продуктовых экранов.
2. Добавить A1 Gallery, если нужен визуальный язык или редакционное направление.
3. Для нового компонента сравнить Component Gallery и Open UI.
4. Выбрать не больше трех направлений и записать, что именно из них проверяется.

### Этап 2. Макет

1. Основной путь: Figma.
2. Penpot использовать, когда нужен бесплатно редактируемый агентом макет.
3. Paper и OpenPencil включать только в отдельном исследовательском контуре.
4. Не переносить макет в код до выбора одного направления владельцем.

### Этап 3. Цвет и токены

1. Leonardo проверяет контраст и строит варианты семантической палитры.
2. Harmonizer и Polychrom помогают визуально проверить OKLCH и APCA.
3. WCAG 2 остается обязательной проверкой; APCA используется дополнительно.
4. Asimonim не становится источником истины сам по себе. Сначала утверждается
   один DTCG-файл, затем Asimonim валидирует и экспортирует его.

### Этап 4. Реализация

1. Использовать существующие HTML, CSS, токены Design OS и компоненты Yalken.
2. Не устанавливать Blueprint, Cloudscape, PatternFly, Ant Design или Tokenami.
3. Не добавлять сетевые обращения, исполняемый MCP или UI-фреймворк в приложение.
4. Context7 использовать только для существующих библиотек и утвержденных API.

### Этап 5. Аудит

1. Запустить UI Craft и технический Audit.
2. После существенной CSS-правки запустить Project Wallace.
3. Провести Heuristic review для пользовательского сценария.
4. Проверить существующие Electron-тесты со снимками и нужные размеры окна.

### Этап 6. Шлифовка и закрытие

1. Проверить длинный текст, пустые состояния, загрузку, ошибки, отключенные элементы и фокус.
2. Сверить макет и реализацию в Browser.
3. Запустить Finalize и классифицировать каждое найденное визуальное отличие.
4. Argos подключать только отдельным CI-контуром, утвержденным владельцем.

## 5. Готовые запросы

### Исследование экрана

```text
Используй Lazyweb как первый источник и A1 Gallery как второй.
Найди реальные примеры этого типа desktop-экрана.
Выдели повторяющиеся решения по структуре, плотности, состояниям и типографике.
Не копируй внешний визуальный стиль целиком.
Предложи три направления, совместимые с каноном Yalken.
```

### Проверка цвета

```text
Используй Leonardo MCP.
Проверь все пары текста и фона по WCAG 2.
Дополнительно покажи APCA, но не используй его как единственный релизный барьер.
Построй тихую семантическую палитру для холста, панели, основного и приглушенного
текста, фокуса, выделения, успеха, предупреждения и ошибки.
```

### CSS-аудит

```text
Запусти Project Wallace на src/renderer/styles.css.
Сравни результат с базовым снимком из матрицы.
Отдели допустимые варианты тем и состояний от случайного визуального дрейфа.
Не исправляй значения автоматически: сначала подготовь группы изменений.
```

## 7. Политика шрифтов

Текущее решение владельца:

- использовать готовые гарнитуры;
- не редактировать и не собирать собственные шрифты;
- не подключать Glyphs MCP, Fontra и шрифтовый CI;
- Circe считать временным кандидатом, а не финальным базовым шрифтом;
- перед добавлением любого файла шрифта фиксировать первоисточник, лицензию,
  право на встраивание в desktop-приложение и поддержку кириллицы;
- отсутствие лицензии или права на встраивание в приложение является стоп-сигналом.

## 8. Паспорт локального подключения

```text
LEONARDO_VERSION: 0.1.0
LEONARDO_MCP_HANDSHAKE: PASS
LEONARDO_TOOLS: generate-theme check-contrast convert-color create-palette
ASIMONIM_VERSION: 0.3.2
ASIMONIM_MCP_HANDSHAKE: PASS
ASIMONIM_TOOLS: search_tokens validate_tokens convert_tokens
ASIMONIM_RUNTIME_NOTE: PACKAGE_REQUIRES_NODE_22_REVERIFY_BEFORE_TOKEN_MIGRATION
PROJECT_WALLACE_VERSION: 9.9.1
PROJECT_WALLACE_FIRST_AUDIT: PASS
PRODUCT_DEPENDENCIES_ADDED: NONE
```

Leonardo и Asimonim установлены глобально и зарегистрированы в Codex. Первый
handshake Asimonim успешно прошел на текущем Node 20, но пакет официально
указывает Node 22 как минимальную версию. Перед реальной миграцией токенов нужно
перейти на Node 22 или повторно подтвердить совместимость.
