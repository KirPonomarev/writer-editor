# Резерв дизайн-инструментов Yalken v1

Статус: внешний исследовательский резерв, не продуктовый runtime

Проверено: 2026-07-20

Назначение: сохранить результаты исследования и вызывать тяжёлые инструменты
только по понятному триггеру. Наличие записи здесь не разрешает добавлять
зависимость, сеть, облако или исполняемый плагин в Yalken Writer.

## 1. Правило применения

1. Сначала использовать уже активный минимальный контур.
2. Из резерва выбирать один инструмент под один подтверждённый пробел.
3. До запуска проверить лицензию, источник, версию, сетевое поведение и область
   доступа.
4. Исследовательские MCP по умолчанию держать выключенными.
5. Результат инструмента является материалом для решения, а не новым каноном.

| Инструмент | Роль |
|---|---|
| Lazyweb | реальные продуктовые референсы и исследовательские отчёты |
| A1 Gallery | визуальные направления, композиция, типографический характер |
| Browser | первоисточники, поведение и визуальная проверка |
| ImageGen | оригинальные растровые визуалы и варианты |

| Figma, Penpot, OpenPencil | один выбранный редактируемый canvas на задачу |

## 3. Установлено или подготовлено, но включается по триггеру

| Инструмент | Для чего | Триггер |
|---|---|---|
| OpenPencil | HTML/CSS в редактируемый `.fig`, токены, lint, экспорт | нужен локальный программируемый макет |
| Penpot MCP | открытый совместный макет | нужен бесплатно редактируемый canvas |
| Paper | быстрый визуальный эскиз | отдельный исследовательский контур |
| Asimonim | проверка и конвертация DTCG | утверждён один DTCG-источник |
| Dembrandt | извлечение дизайн-ДНК сайта | разрешённый сайт и нужен DOM/CSS-слепок |
| 21st Reference | справочный компонентный код | визуальный контракт уже выбран |
| Chrome DevTools MCP | глубокая диагностика браузерной страницы | встроенного Browser недостаточно |
| Playwright Reference | отдельный браузерный стенд | нужен воспроизводимый сценарий |
| shadcn Reference | только справочные паттерны | утверждена совместимая компонентная модель |

## 4. Ближайшие внешние пилоты

| Инструмент | Для чего | Ограничение |
|---|---|---|
| Open Design | быстрые HTML-артефакты и альтернативные направления | не устанавливать: релиз `0.15.1` не прошёл локальную строгую проверку подписи |
| Impeccable detector | детерминированный поиск AI-slop и UI-дефектов | только detector; не создавать второй `DESIGN.md` |
| SiteOne Crawler | карта разрешённого сайта, Markdown, снимки и аудит | отдельный CLI, не MCP по умолчанию |
| MCP Inspector | проверка MCP до подключения к Codex | запускать перед регистрацией нового сервера |

## 5. Резерв по специальным задачам

| Инструмент | Когда может понадобиться |
|---|---|
| Crawl4AI | большой многостраничный сайт нужно превратить в смысловую карту |
| Browsertrix | нужна доказательная WARC/WACZ-копия разрешённого сайта |
| ArchiveBox, SingleFile, Zimit, HTTrack | нужен долговременный или автономный архив сайта |
| SiteOne | нужен практический offline-клон, Markdown и технический аудит |
| Docling | большая книга, сложный PDF, OCR или lossless document JSON |
| MarkItDown | нужен лёгкий пакетный перевод документов в Markdown |
| Xberg | нужны точные bbox, provenance и единый разбор многих форматов |
| Open Notebook, Zotero MCP | появляется постоянная исследовательская библиотека |
| Webwright | браузерное исследование должно стать повторяемым скриптом |
| OmniParser | DOM недоступен, интерфейс нарисован через canvas или WebGL |
| Lost Pixel, Argos | появился стабильный набор эталонных web-экранов |
| Unlighthouse, sitespeed.io | нужен обход многих web-страниц и история performance |
| IBM Equal Access | нужен второй независимый accessibility engine |
| rrweb, OpenReplay | появились реальные пользователи и согласованный session replay |
| ToolHive | подключается несколько недоверенных MCP с разными правами |
| Agent Scan | нужно проверять сторонние навыки и MCP в изоляции |
| ContextForge | появляется командный gateway с большим числом серверов |
| Langfuse | появляется постоянный агентный сервис и нужна трассировка решений |

## 6. Наблюдать, но не интегрировать сейчас

| Инструмент | Причина |
|---|---|
| A2UI, MCP Apps, OpenUI | протоколы и host-поддержка ещё развиваются |
| json-render | полезна идея allowlisted JSON UI, но пакет не нужен текущему runtime |
| ZSeven OpenPencil | молодой параллельный design-as-code canvas |
| Open Lovable, Dyad, bolt.diy | отдельные app-builder песочницы с большим контуром доверия |
| Onlook, Puck, React Grab | React-ориентированный рабочий процесс |
| Agentation | React 18+, desktop-only и PolyForm Shield |
| JSON Forms, Orval | нужны только при JSON Schema/OpenAPI и совместимом runtime |
| Figbridge, Plumb | молодые неофициальные Figma-мосты |

## 7. Стоп-лист для текущего Yalken

- Не добавлять исследовательские инструменты в `package.json` приложения.
- Не запускать захват сайтов без разрешения и зафиксированной цели.
- Не переносить чужие тексты, изображения, логотипы, шрифты и код без лицензии.
- Не включать больше одного design canvas на одну задачу.
- Не писать `site-transmuter-mcp`, `book-to-product-mcp` и
  `design-evaluator-mcp`, пока повторяемая ручная цепочка не доказала пользу.
- Не заменять канон результатом автоматического анализа или генерации.

## 8. Порядок активации резерва

1. Назвать пробел и ожидаемый артефакт.
2. Выбрать самый узкий инструмент.
3. Проверить первоисточник, лицензию и текущую версию.
4. Установить снаружи продукта и выдать минимальные права.
5. Выполнить один пилот на копии или разрешённом источнике.
6. Сравнить с существующим минимальным контуром.
7. Оставить `GATED`, повысить до `ACTIVE` или удалить.

## 10. Интеграционный срез 2026-07-20

| Инструмент | Статус | Проверка и способ применения |
|---|---|---|
| SiteOne Crawler `2.5.1` | `ACTIVE_EXTERNAL` | установлен через официальный Homebrew tap; CLI и версия проверены |
| Impeccable `3.2.1` | `GATED_EXTERNAL` | запускается из npm-кэша через Node 24; тестовый detector-проход по renderer выполнен, зависимость в проект не добавлена |
| MCP Inspector `1.0.0` | `GATED_EXTERNAL` | CLI загружен в npm-кэш и проверен; запускать вручную перед регистрацией нового MCP |
| OpenPencil `0.13.2` | `ACTIVE_AFTER_RESTART` | CLI и MCP уже установлены; сервер включён во внешней конфигурации Codex, текущей сессии требуется перезапуск |
| Open Design `0.15.1` | `BLOCKED_SIGNATURE` | SHA-256 совпал с релизом, но приложение не прошло `codesign --verify --deep --strict`; установка остановлена |
| Figma | `ACTIVE` | официальный коннектор Codex установлен; авторизация и доступ к плану Starter подтверждены через `whoami` |
| Penpot MCP | `SESSION_PENDING` | плагин установлен; для работы нужно открыть Penpot, запустить плагин и поднять локальный мост |

Предупреждения Impeccable не являются задачами на автоматическое исправление:
детектор может давать ложные срабатывания и всегда требует сопоставления с каноном,
реальным рендером и ручной проверкой.
